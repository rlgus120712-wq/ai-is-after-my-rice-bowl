# 🍚 AI가 내 밥그릇을 노린다
### AI 시대, 개발자가 살아남기 위한 지식 허브

---

## 1. 왜 이 프로젝트를 만들었는가

AI 기술의 발전 속도는 전례 없이 빠릅니다.

ChatGPT가 등장한 지 2년 만에 GPT-4, Claude, Gemini가 나왔고, 코드 자동완성을 넘어 코드 전체를 생성하는 시대가 되었습니다. 개발자들 사이에서는 이런 말이 나옵니다.

> **"AI가 내 일을 대체하는 건 시간문제 아닌가?"**

하지만 현실은 다릅니다. AI는 도구이고, 그 도구를 **이해하고 활용하는 개발자**와 그렇지 못한 개발자 사이의 격차가 벌어지고 있습니다.

문제는 **정보가 너무 많고, 너무 빠르게 변한다**는 것입니다.

- 매일 수십 개의 AI 관련 논문과 뉴스가 쏟아집니다
- 어떤 기술이 실제로 현업에서 쓰이는지 판단하기 어렵습니다
- 개발자들이 서로 경험을 나누고 검증할 공간이 부족합니다

이 프로젝트는 그 문제를 해결하기 위해 만들어졌습니다.

---

## 2. 무엇을 만들었는가

**AI가 내 밥그릇을 노린다**는 두 가지 기능을 중심으로 합니다.

### 기능 1. AI 트렌드 자동 수집

개발자가 직접 뉴스를 찾아다니지 않아도, 지금 AI 업계에서 어떤 기술이 주목받고 있는지 한눈에 볼 수 있습니다.

### 기능 2. 개발자 생존 블로그

단순한 뉴스 소비에서 그치지 않고, 개발자들이 직접 자신의 경험과 인사이트를 남길 수 있는 공간입니다. AI 시대를 어떻게 헤쳐나가고 있는지, 어떤 기술을 왜 배웠는지를 공유합니다.

---

## 3. 왜 HackerNews와 Dev.to인가

수많은 정보 소스 중에서 이 두 곳을 선택한 이유가 있습니다.

### HackerNews
- 전 세계 개발자, 스타트업 창업자, 투자자들이 모이는 커뮤니티
- **실제로 현업에서 중요하게 여기는 기술**이 올라옴
- 커뮤니티 투표로 걸러지기 때문에 노이즈가 적음
- 새로운 AI 프레임워크, 논문, 도구가 가장 먼저 공유되는 곳

### Dev.to
- 개발자가 직접 작성하는 실전 경험 중심의 아티클
- **"이걸 어떻게 썼다"는 실용적인 내용**이 많음
- AI 툴을 실제 업무에 적용한 후기, 튜토리얼이 활발하게 올라옴

두 소스를 함께 쓰는 이유는 **업계 흐름(HN) + 실전 활용법(Dev.to)** 을 동시에 커버하기 위해서입니다.

## 4. AI의 미래와 개발자의 역할

### AI가 바꾸고 있는 것

AI는 반복적이고 예측 가능한 작업을 빠르게 대체하고 있습니다.

- 보일러플레이트 코드 작성
- 간단한 버그 수정
- 문서화, 테스트 케이스 생성

### 그렇다면 개발자는 무엇을 해야 하는가

AI가 대체하기 어려운 영역이 있습니다.

- **문제를 정의하는 능력** — 무엇을 만들어야 하는지 판단하는 것
- **시스템을 설계하는 능력** — 어떻게 구성할지 아키텍처를 그리는 것
- **AI 결과물을 검증하는 능력** — AI가 생성한 코드가 맞는지 판단하는 것
- **도메인 지식** — 비즈니스 맥락을 이해하고 기술적 판단을 내리는 것

앞으로의 개발자는 **AI를 지시하고 검토하는 역할**로 이동합니다. 이를 위해 MCP, RAG, Agent, 프롬프트 엔지니어링 같은 AI 활용 기술이 필수 역량이 되고 있습니다.

### AI 활용도의 변화 방향

```
현재
  코드 자동완성 → 코드 생성 → 기능 단위 구현

가까운 미래
  요구사항 입력 → AI가 초안 작성 → 개발자가 검토/수정

먼 미래
  비즈니스 목표 입력 → AI가 설계·구현·테스트 → 개발자가 의사결정
```

이 흐름에서 살아남으려면 AI 기술 자체를 이해하고, 동료 개발자들의 실전 경험을 빠르게 흡수해야 합니다.

---

## 5. 이 블로그가 필요한 이유

### 개발자 간 지식 공유의 문제

지금까지 개발자들은 블로그, 유튜브, 트위터 등 여러 채널에 흩어져 있었습니다.

- AI 트렌드를 따로, 블로그를 따로 찾아야 함
- 검증되지 않은 정보가 많음
- 한국 개발자들의 실전 경험이 한데 모이는 공간이 없음

### 이 플랫폼이 제공하는 것

| 역할 | 내용 |
|------|------|
| **정보 수집** | AI 업계 동향을 매일 자동으로 모아 제공 |
| **지식 축적** | 개발자들이 직접 경험과 인사이트를 남김 |
| **의사소통** | 같은 고민을 가진 개발자들이 서로의 글을 읽고 배움 |
| **동기 부여** | "나만 뒤처지는 게 아니다"는 공감대 형성 |

단순한 뉴스 피드가 아니라, **개발자가 AI 시대를 함께 탐색하는 공간**입니다.

## 6. 서비스 구조 요약

```
매일 오전 9시
HackerNews + Dev.to → AI 트렌드 자동 수집 → 트렌드 페이지 업데이트

개발자
GitHub 로그인 → 마크다운으로 글 작성 → 블로그 발행
→ 다른 개발자들이 읽고 인사이트 공유
```

---

## 7. 기술적 선택의 이유

기술 선택도 모두 이 서비스의 목적에서 출발했습니다.

| 선택 | 이유 |
|------|------|
| **Next.js ISR** | 트렌드·블로그 목록은 자주 바뀌지 않으므로 캐시해서 빠르게 제공 |
| **Supabase** | 별도 서버 없이 DB + 인증을 빠르게 구성, 오픈소스 기반 |
| **GitHub OAuth** | 개발자 커뮤니티에서 가장 자연스러운 로그인 방법 |
| **Vercel Cron** | 서버 운영 없이 매일 트렌드 수집을 자동화 |
| **Markdown** | 개발자에게 익숙한 글쓰기 방식 |

## 8. 앞으로의 방향

이 프로젝트는 시작점입니다. 앞으로 확장 가능한 방향은 다음과 같습니다.

- **AI 요약 기능** — 수집된 기사를 GPT API로 자동 요약해서 제공
- **태그 기반 필터** — 관심 기술(MCP, RAG, Agent 등)별로 트렌드 필터링
- **댓글 / 좋아요** — 개발자 간 상호작용 강화
- **뉴스레터** — 주간 AI 트렌드 요약 이메일 발송
- **트렌드 분석** — 어떤 기술이 얼마나 언급되는지 시각화

## 9. 마무리

AI가 개발자의 밥그릇을 위협하는 건 사실입니다.

하지만 **AI를 이해하고, 함께 쓰고, 서로 배우는 개발자**는 오히려 더 강해집니다.

이 플랫폼은 그 과정을 돕기 위해 만들어졌습니다.

> "AI 시대에 살아남는 개발자는 AI를 두려워하는 개발자가 아니라, AI를 가장 잘 다루는 개발자다."
